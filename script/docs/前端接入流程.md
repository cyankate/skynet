# 客户端接入服务端协议说明

本文用于指导前端同学在 Unity 客户端工程中接入当前服务端协议与网络链路。

## 1. 接入范围（必须与可选）

- 必须：
  - Lua 协议文件与运行时依赖
  - C# TCP 分帧收发
  - C# 与 Lua 的桥接
- 可选：
  - 协议字段校验（开发期建议开启）
  - 登录演示流程（可按项目替换）

## 2. 客户端目录放置规范

以下为推荐目录（与当前示例一致）：

```text
Assets/
  Plugins/
    x86_64/
      lpeg.dll
      sproto/
        core.dll
  StreamingAssets/
    Lua/
      lualib/
        sproto.lua
        sprotoparser.lua
      script/
        protocol/
          proto.lua
          proto_builder.lua
  Scripts/
    Network/
      TcpFrameTransport.cs
    Lua/
      ILuaBridge.cs
      XLuaBridge.cs
    Threading/
      MainThreadDispatcher.cs
    Skynet/
      SkynetClientBootstrap.cs
```

## 3. 客户端需要落地的文件

- 直接复制到前端项目：
  - `StreamingAssets/Lua/script/protocol/proto.lua`
  - `StreamingAssets/Lua/script/protocol/proto_builder.lua`
- 参考实现（建议复制后按项目命名调整）：
  - `TcpFrameTransport.cs`（TCP 分帧收发）
  - `MainThreadDispatcher.cs`（网络线程回调切主线程）
  - `ILuaBridge.cs`（Lua 桥接接口）
  - `XLuaBridge.cs`（xLua 实现）
  - `SkynetClientBootstrap.cs`（启动与串联入口）

## 4. Lua 与原生依赖

- Lua 文件：
  - `sproto.lua`
  - `sprotoparser.lua`
- 原生模块：
  - `sproto.core`
  - `lpeg`

推荐放置路径（Unity）：

- `Assets/Plugins/x86_64/lpeg.dll`
- `Assets/Plugins/x86_64/sproto/core.dll`

说明：

- 目录中的 `x86_64` 需要与 `XLuaBridge` 的 `pluginsSubdir` 配置一致。
- 如果你的项目使用其他平台目录（如 Android/iOS），按该平台插件规范放置同名模块。

请确保插件路径与 `XLuaBridge` 中 `pluginsSubdir` 配置一致。

## 5. Lua 路径配置要求

前端需要确保 Lua 可以找到 `protocol/proto.lua`。  
如果前端直接使用当前仓库里的 `XLuaBridge`，这一步通常已内置，可跳过。

如果前端使用自己的 Lua 初始化逻辑，请在启动时设置 `package.path`，示例：

```lua
package.path = package.path
    .. ";Assets/StreamingAssets/Lua/script/?.lua"
    .. ";Assets/StreamingAssets/Lua/script/?/init.lua"
    .. ";Assets/StreamingAssets/Lua/script/?/?.lua"
```

配置完成后应满足：

- `require "protocol.proto"` 可以成功加载

## 6. 常见问题

- `module 'protocol.proto' not found`
  - 检查 `package.path` 是否包含 `script/?/?.lua`
- `attempt to call a nil value (global 'lpeg')`
  - 缺少 `lpeg` 原生库
- 解包失败或协议号错乱
  - 客户端与服务端 `protocol` 版本不一致
